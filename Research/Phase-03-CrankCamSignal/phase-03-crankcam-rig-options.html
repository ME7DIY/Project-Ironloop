Project IRONLOOP

Crank/Cam Rig Options

Phase 03 Research Log - March 5, 2026

What Triggered This Note

Community discussion surfaced a potentially faster hardware path for Phase 03 crank/cam validation by combining proven stim tooling with stronger instrumentation.

Community Input - Validate Before Adoption

Key Hardware Concepts

  • Use ArduStim + Mega2560 as an alternate crank/cam signal source.
  • Use a 16-channel Saleae for broad timing visibility and capture.
  • Use an ECU-footprint stim baseboard for cleaner harness integration.
  • Add modular bench blocks: digital pot mounts, power distribution, CAN hub, and a Pi node for candump.

Why This Could Be Game Changing

  1. Potentially faster crank/cam bring-up than pure custom path from day one.
  2. Higher measurement confidence through full-channel capture strategy.
  3. Cleaner, repeatable bench wiring architecture that scales to future ECUs.
  4. Reduced debug time when tooth timing issues appear.

AC / VR Sensor Simulation Question

Open question from discussion: how to generate realistic AC VR behavior for conditioner validation.

Options To Investigate

  • Dedicated VR driver circuitry (variable amplitude/frequency, differential output).
  • Physical wheel + pickup rig for high-realism signal generation.
  • Function generator based sweep testing for controlled characterization.

Recommendation: keep this as a dedicated validation workstream, not a blocker for initial digital crank/cam success criteria.

Recommended Evaluation Plan

  1. Keep Pico path as baseline implementation.
  2. Prototype ArduStim in parallel as comparison track.
  3. Capture side-by-side timing data with 16-channel logic analysis.
  4. Score each approach on jitter, phase stability, bring-up effort, and maintainability.
  5. Select Pico-only, ArduStim-only, or hybrid architecture based on measurements.

Immediate Actions

  • Add this option set to Phase 03 planning checklist.
  • Define objective waveform pass/fail metrics before testing.
  • Document whether VR simulation is Phase 03 required scope or deferred validation.
