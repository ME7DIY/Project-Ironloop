Project IRONLOOP

Phase 2 Minimum Signals Research

Research Reference - March 6, 2026

Objective

Define the smallest practical signal set needed for Bosch ME7 bench bring-up in Phase 2 so the ECU powers, reads plausible inputs, and avoids immediate diagnostic shutdown behavior. This page is scoped to minimum viable emulation, not full closed-loop realism.

Minimum Required Inputs

These are the highest-priority channels to emulate first:

Signal Why It Matters Bridge Source Output Path
TPS / throttle angle Driver/load plausibility and idle/load transitions. engine.throttle MCP4728 DAC voltage
MAP / manifold pressure Core load determination and fuel base calculation. engine.manifold_pressure MCP4728 DAC voltage
IAT intake temp Air-density compensation and fallback logic checks. engine.intake_temp_c Digital pot or resistance emulation
ECT coolant temp Warm-up behavior, enrichment, and protection states. Mapped engine thermal model value Digital pot or resistance emulation
Narrowband O2 Lambda correction path and fault suppression during checks. engine.exhaust_o2 mapped to NB curve MCP4728 DAC voltage
System voltage awareness Several corrections and plausibility gates depend on battery conditions. Bench PSU measured value Software correction path

Target Update Rates and Stability

  • Bridge ingest target: 50-100 Hz steady processing.
  • Signal output update target: >= 50 Hz with bounded step size on fast-changing channels.
  • Temperature channels should be slew-limited to avoid unrealistic jumps that trigger plausibility checks.
  • O2 narrowband should use sigmoid/filtered switching behavior, not hard one-cycle toggles.
Practical rule: smooth enough to avoid false faults, responsive enough that scanner live data tracks expected changes within ~100-200 ms.

Known Calibrated Behaviors Already Validated in Project

  • ECT/IAT resistance saturation issue was corrected (100 ohm pinning fixed).
  • Config-based remap and limits already implemented in bridge configuration.
  • Narrowband O2 modes available: step and sigmoid; sigmoid is preferred.
  • Software-path validation showed plausible ranges for O2 and IAT resistance response.

Phase 2 Bring-Up Sequence

  1. Start with static-safe values on all minimum channels and confirm ECU communication over bench OBD.
  2. Enable TPS + MAP dynamics only; verify readings follow expected direction and no severe DTC storm.
  3. Add IAT/ECT thermistor emulation with conservative slew limits.
  4. Enable narrowband O2 emulation and verify lambda-related behavior remains stable.
  5. Capture and review logs for all output channels plus diagnostic codes after each step.

Minimum Pass Criteria

  • ECU powers consistently on bench and remains communicative for repeated sessions.
  • Live data shows coherent movement for TPS, MAP, IAT, ECT, and O2 channels.
  • No persistent critical sensor implausibility faults on minimum-signal channels.
  • Bridge can run at target rate and log outputs without dropped processing loops.
