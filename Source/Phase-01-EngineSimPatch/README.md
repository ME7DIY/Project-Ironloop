## Overview

Fork the official ange-yaghi/engine-sim C++ project and add a local UDP or TCP socket broadcast. Every simulation frame, the engine state (RPM, throttle, MAP, MAF, ECT, lambda, etc.) is serialized to JSON and emitted over the network.

This gives the Python bridge a live feed of what the virtual engine is doing, enabling the closed-loop simulation.

## Key Variables to Export

These state variables must be extracted from the engine-sim simulation loop and broadcast:

| Variable | Type | Source | Units |
|----------|------|--------|-------|
| **rpm** | float | Engine object | RPM |
| **throttle_position** | float | Crankshaft | 0.0–1.0 |
| **manifold_pressure** | float | Intake manifold | kPa |
| **mass_air_flow** | float | Air intake | g/s or kg/s |
| **coolant_temp** | float | Engine coolant | °C |
| **intake_air_temp** | float | Intake air | °C |
| **lambda** | float | Combustion | 0.8–1.2 (stoich = 1.0) |
| **cam_angle** | float | Cam shaft | Degrees |
| **crank_angle** | float | Crank shaft | Degrees |
| **fuel_flow** | float | Injectors | ml/s or cc/s |
| **ignition_advance** | float | Calculated | Degrees BTDC |

## Socket Protocol Design

### Decision: UDP vs TCP?

**UDP:**
- Faster (no handshake)
- Stateless
- Some packets may be lost (acceptable for real-time gauge data)
- Better for 60+ Hz broadcasts

**TCP:**
- Guarantees delivery
- Adds overhead
- Better for critical data
- Slower acknowledgment

**Recommendation:** Start with **UDP on localhost:5555**. Real-time frame data can tolerate occasional packet loss.

### Example JSON Payload

```json
{
  "timestamp_ms": 1234567890,
  "frame_number": 42000,
  "engine": {
    "rpm": 3500.5,
    "throttle_position": 0.75,
    "manifold_pressure": 85.2,
    "coolant_temp": 92.0,
    "intake_air_temp": 28.5,
    "lambda": 0.98
  },
  "fuel": {
    "injector_pw_ms": 4.2,
    "fuel_flow_cc_s": 8.5
  },
  "ignition": {
    "advance_deg_btdc": 18.5
  },
  "timing": {
    "cam_angle": 45.0,
    "crank_angle": 120.0
  }
}
```

## Implementation Notes

### Where to Add Socket Code?

Inside the main engine-sim loop (likely in `Engine::update()` or frame handler):

1. Serialize `engine_state` to JSON
2. Send to UDP socket `127.0.0.1:5555`
3. Keep latency < 16ms (for 60 Hz)

### Socket Library Options

- **Cross-platform:** Boost.Asio (if available), Asio standalone
- **Windows-only:** Winsock2 (easier setup)
- **Linux-only:** POSIX sockets

### JSON Library

- **Recommended:** nlohmann/json (header-only, easy to integrate)
- **Alternative:** RapidJSON (faster, lower latency)
