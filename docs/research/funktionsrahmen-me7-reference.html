Section 01

Corpus Overview

The source folder includes broad ME7.1 logic coverage: air charge, speed/crankshaft processing, fuel, lambda emissions, ignition, knock, torque management, transmission, VVT camshaft control, and diagnostics. For IRONLOOP, the highest-value portions are where sensor conversion, trigger timing, and fault-path behavior intersect with bench emulation.

Largest source files are fuel injection (43 sections), diagnostics (37 sections), emissions lambda (26 sections), and air charge (25 sections). These together define most of the practical behavior we need to mimic or observe.

Section 08

IRONLOOP Extraction Plan

This is the practical subset to pull into active project docs and code comments first:

  • 1) Sensor core pack: air charge + temperatures + fallback thresholds (Phase 2).
  • 2) Trigger and sync pack: crank/cam acceptance windows and timeout behavior (Phase 3).
  • 3) Fuel/lambda pack: narrowband behavior and correction dependencies (Phase 2 to 4).
  • 4) Torque interaction pack: requested vs permitted torque structure (Phase 4+).
  • 5) Diagnostic gate pack: fault/debounce patterns to prevent false negatives during bench bring-up.
Recommended workflow: convert each pack into a small IRONLOOP-specific reference sheet with signals, equations, fault gates, and bench validation criteria.
